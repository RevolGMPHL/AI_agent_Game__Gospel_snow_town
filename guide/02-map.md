## 室内场景架构

### 统一布局规则
- **y=0**：北墙（不可走）
- **x=0**：西墙（不可走）
- **x=width-1**：东墙（不可走）
- **y=height-1**：南墙，中间2格为出口门（`doorX-1` ~ `doorX`）
- **家具区域**：在 `isSolid()` 中标记为不可走
- **可走区域**：扣除墙壁和家具后的空地

### 多座位系统（`_pickIndoorSeat`）
每个室内场景定义了多个 NPC 可站立的座位位置（`INDOOR_SEATS`），避免所有 NPC 堆叠在同一点：

| 场景 | 座位数量 | 典型位置 |
|------|---------|---------|
| `dorm_a` | 7 | 火盆旁左/右、简易桌旁、杂物桌旁、各床边 |
| `dorm_b` | 7 | 火盆旁、火盆前、桌旁、各床边 |
| `medical` | 5 | 各病床旁、药柜旁、诊疗台旁 |
| `warehouse` | 5 | 各物资区旁、仓库中央 |
| `workshop` | 6 | 工作台前、发电机旁、工具架旁、建材堆旁、无线电台旁 |
| `kitchen` | 6 | 灶台旁、食材架旁、餐桌4个座位 |

### 进门机制（`_enterIndoor`）
NPC 进入室内采用**直接传送到座位**方式，不经过门口中转：

```
NPC 到达建筑门口（village 场景中 dist <= 5）
→ _enterIndoor(targetScene, game)
  → _pickIndoorSeat(targetScene, game)  选择未被占用的座位
  → 精确设置像素坐标 this.x = x * TILE; this.y = y * TILE;
  → scheduleReached = true
  → _indoorEntryProtection = 3  (3秒进屋保护期)
```

⚠️ **重要**：不使用 `_teleportTo`（有 ±1.5格随机偏移），直接设置精确坐标。

### 出门机制（`_walkToDoorAndExit`）
NPC 从室内走到门口后传送到 village 的建筑入口位置。所有跨室内场景导航必须经 village 中转。

### 室内场景 PNG 图片
`asset/indoor/` 目录包含每个室内场景的布局图（原始尺寸 + 4x放大版），可用于 AI 重绘替换：
- 绿色小圆点 = NPC 可站立座位
- 底部中间2格 = 出口门
- 生成脚本：`generate_indoor_maps.py`
